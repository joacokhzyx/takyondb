# Contributing to TakyonDB

Thank you for your interest in contributing to TakyonDB! To maintain our extreme performance and safety standards, all contributors must adhere to these guidelines.

## Code of Conduct
We expect all collaborators to act professionally, communicate respectfully, and prioritize technical excellence.

## Git Workflow
1.  Fork the repository and create a feature branch (`git checkout -b feature/amazing-performance`).
2.  Follow the guidelines in [STYLEGUIDE.md](file:///C:/Users/Alumnos/Downloads/TakyonDB/docs/STYLEGUIDE.md) strictly.
3.  Add tests to verify your implementation.
4.  Ensure that `zig fmt` passes for Zig core changes and standard linting scripts pass for TypeScript changes.
5.  Submit a Pull Request with a clear description of the problem solved and performance metrics where applicable.

## Contributor License Agreement (CLA)

TakyonDB operates under a **Dual-Licensing Model**:
- **GNU AGPLv3** for open-source and non-commercial users.
- **Commercial License ($10 / month)** for enterprise and proprietary software vendors.

In order for us to legally offer the Commercial License and maintain this funding model, we must own or have full rights to 100% of the codebase. 

Because of this, **all contributors must sign our Contributor License Agreement (CLA)** before we can merge any Pull Requests.

By signing the CLA, you grant TakyonDB the right to distribute your contributions under both the AGPLv3 and our Commercial License. You still retain the original copyright to your own code. When you open your first Pull Request, an automated CLA bot will comment on the PR with instructions on how to sign the agreement digitally.
